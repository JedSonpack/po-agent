# Code Review

Use this skill when asked to review code.

## Steps

1. **Read fully** — read the entire file before commenting.
2. **Check correctness** — trace edge cases: empty input, None, off-by-one.
3. **Check style** — naming, dead code, duplication.
4. **Report** — group findings by severity (bug / style / nit). Cite file:line.

## Severity

- **bug**: wrong output, crash, security hole.
- **style**: readability, naming, dead code.
- **nit**: minor, optional.
